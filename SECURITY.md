# Security Policy

## Supported Versions

| Version | Supported |
|---------|-----------|
| 1.x     | ✓         |

## Reporting a Vulnerability

Please report security vulnerabilities by opening a
[GitHub Security Advisory](https://github.com/geronimo-iia/soa-rs/security/advisories/new).

**Do not open a public issue.**

You should receive an acknowledgment within 48 hours. A fix will be
prioritized based on severity and released as a patch version.

## Scope

soa-rs is a `no_std` data-structure library. It does not make network
requests, store credentials, or execute user-supplied code. The main
attack surface is:

- Unsafe pointer arithmetic in `SoaRaw` implementations generated by the
  derive macro
- Memory layout assumptions in the single-allocation SoA model

Dependency vulnerabilities are tracked via `cargo audit` in CI and
Dependabot alerts.
